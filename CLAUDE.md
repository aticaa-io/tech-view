# Tech View — Claude Code 작업 가이드

## 프로젝트 개요

테크 제품 분석 다국어 콘텐츠 프로젝트.
Claude Code + 웹 리서치로 데이터 수집 → 분석 → 리포트 → 소셜미디어 발행.

## 디렉터리 역할

| 디렉터리 | 역할 | 수정 빈도 |
|----------|------|----------|
| `methodology/` | 리포트 템플릿, 분석 프레임워크 | 낮음 (표준) |
| `docs/{brand}/` | 브랜드별 배경 지식 (KB) | 중간 |
| `result/` | 완성 리포트 (한/영 쌍) | 높음 (매주 추가) |
| `output/x/` | X 스레드용 변환 | 높음 (발행 시) |

## 리포트 작성 규칙

### 반드시 따를 것
1. **`methodology/01-report-template.md`를 먼저 읽고** 섹션 구조를 준수할 것
2. **가격 다중 병기**: 현지 통화 + $ + ₩
   - 미국: `$1,199 (₩1,738,550)`
   - 한국: `₩1,738,000 ($1,199)`
   - 일본: `¥189,800 ($1,265, ₩1,834,350)`
3. **환율 근사치**: $1 ≈ ₩1,450 / ¥150 (2026년 3월 기준)
4. **한/영 쌍으로 생성**: `{번호}-{주제}-ko.md` + `{번호}-{주제}-en.md`
5. **영문은 번역이 아닌 독립 작성**: 글로벌 독자 시점으로 재작성

### 8개 필수 섹션
0. **Product Verdict** (🟢🟡🟠🔴 신호등 등급, 요약보다 위에 배치)
1. 제목 & 메타데이터 (데이터 기준일, 출처, 분석 대상)
2. 요약 (3–5개 핵심 포인트, 숫자 포함, 볼드 키워드)
3. 분석 본문 (스펙 테이블, 벤치마크 ASCII 차트, 가격 비교)
4. 인사이트 (5개 내외, "왜?"에 대한 해석, 반직관적 발견 우선)
5. 용어집 (7–12개, 초보자도 이해 가능)
6. 레퍼런스 (데이터 출처, 도구, 면책 조항)
7. **Reality Check** (출시 후 실제 리뷰/벤치마크와 사전 분석 비교)

### 마크다운 표기
- **범위 표시**: `~` 대신 `–` (en dash) 사용 → `2025–2026`, `100–120fps`
  - `~`는 "약(approximately)" 의미로만 사용 → `~$999`, `~150g`

### 시각화
- ASCII 바 차트: `█` (양수) / `▓` (음수), 최대값 40칸 기준
- 벤치마크 비교: 수치 + 바 차트 병행
- 비교 테이블: 우세한 쪽에 ★ 표시

## 분석 프레임워크 (methodology/ 참조)

| 프레임워크 | 용도 | 예시 |
|-----------|------|------|
| 제품 H2H 매치업 | 경쟁 제품 1:1 비교 | iPhone 17e vs Galaxy S26 |
| 칩 벤치마크 비교 | 프로세서/GPU 성능 비교 | M5 Pro vs Snapdragon X Elite |
| 가격 대비 성능 | 가성비 분석 | 시장별 가격 × 벤치마크 |
| 제품 라인업 분석 | 브랜드 전략 해석 | Apple 2026 라인업 포지셔닝 |
| 세대별 진화 | 전작 대비 개선점 | iPhone 16e → 17e 변화 |

## output/ 빌드 시스템

소스 문서(docs/, result/)를 플랫폼에 최적화된 형태로 변환.

### 디렉터리 구조
```
output/
└── x/
    └── {번호}_{날짜}_{주제}/
        ├── thread-ko.md
        └── thread-en.md
```

### X 스레드 변환 규칙
- **테이블 사용 금지** — X 아티클에서 마크다운 테이블 미지원
- **ASCII 바 차트로 대체** — 시각적 비교 (█▊▌▎ 블록 문자)
- **7트윗 내외** — `## 1/7`, `## 2/7` 형식으로 구분
- **`---`로 트윗 경계 표시**
- 한/영 독립 작성 (번역이 아닌 재작성)
