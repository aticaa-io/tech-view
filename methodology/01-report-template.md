# 테크 리포트 템플릿 가이드

## 공통 구조

모든 테크 분석 리포트는 아래 8개 섹션을 반드시 포함한다.

### 0. Product Verdict (제품 판단)

리포트의 **맨 첫 화면**에 배치. 독자가 3초 안에 핵심 결론을 파악할 수 있도록.

```markdown
## Product Verdict

🟢 **Strong Buy**: iPhone 17 Pro — A19 Pro, 48MP 트리플, $999부터. 가장 완성도 높은 올라운더
🟡 **Buy**: iPhone 17e — A19 칩, $599 가성비 최강. 대부분의 사용자에게 충분
🟠 **Wait**: Galaxy S26 Ultra — 스냅드래곤 X Elite, 스펙 최강이나 가격 $1,399. 할인 대기 추천
```

#### 신호등 등급 체계

| 등급 | 아이콘 | 기준 |
|------|--------|------|
| **Strong Buy** | 🟢 | 성능 + 가성비 + 완성도 모두 우수 |
| **Buy** | 🟡 | 대부분 만족, 소수 아쉬운 점 존재 |
| **Wait** | 🟠 | 좋지만 가격 하락/다음 세대 대기 권장 |
| **Skip** | 🔴 | 가성비 낮거나 더 나은 대안 존재 |

### 1. 제목 & 메타데이터

```markdown
# [제품/비교 주제] 분석 리포트

> **데이터 기준일**: YYYY-MM-DD
> **데이터 출처**: [공식 스펙, 벤치마크 소스]
> **분석 대상**: [제품명 나열]
```

### 2. 요약 (Summary)

- 3–5개 핵심 포인트를 불릿으로 정리
- 스펙 수치(%, 점수, 가격)를 반드시 포함
- **볼드**로 핵심 키워드 강조

### 4. 인사이트 (Insight)

- 5개 내외의 분석적 통찰
- **"왜?"에 대한 해석** 제공
- 반직관적이거나 놀라운 발견을 우선 배치

### 5. 용어 (Glossary)

- 7–12개 기술 용어를 테이블로 정리
- 비전문가도 이해할 수 있는 수준

### 6. 레퍼런스 (References)

- 공식 스펙 출처 (Apple/Samsung/Qualcomm 공식)
- 벤치마크 출처 (Geekbench, AnTuTu 등)
- 가격 출처 (공식 스토어 기준)
- 면책 조항

### 7. Reality Check (현실 검증)

출시 후 실제 리뷰/벤치마크가 사전 분석과 일치하는지 검증.

```markdown
## Reality Check

> 📊 출시 전 분석 vs 실제 결과

| 항목 | 사전 예측 | 실제 결과 | 판정 |
|------|----------|----------|------|
| Geekbench 싱글 | ~3,400 | 3,380 | ✅ 적중 |
| 배터리 수명 | 18h+ | 16.5h | ❌ 미달 |
| 발열 | 양호 예상 | 쓰로틀링 보고 | ⚠️ 주의 |
```
